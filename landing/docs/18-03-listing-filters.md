# Фильтрация в листингах каталога и поиска

> Последнее обновление: 2026-03-18

## Проблема

На страницах каталога (`/stil/portret/`, `/promty-dlya-foto-devushki/`) и в поиске (`/search?q=...`) пользователь видит сотни карточек без возможности уточнить выборку. Нужна фильтрация по ключевым измерениям.

## Где работает

- **Страницы каталога** — tag-страницы `/stil/*`, `/promty-dlya-foto-*/*`, и т.д.
- **Поиск** — `/search?q=...`

## Параметры фильтрации

По убыванию приоритета:

| # | Измерение | UI-название | Примеры значений | Кол-во | Реализация |
|---|-----------|------------|------------------|--------|------------|
| 1 | `audience_tag` | Кто на фото | Девушка, Мужчина, Пара, Семья, Дети | ~24 | Чипсы |
| 2 | `style_tag` | Стиль | Портрет, Чёрно-белое, 3D, Аниме, Реалистичное | ~45 | Чипсы |
| 3 | `occasion_tag` | Событие | День рождения, Свадьба, Новый год, 8 марта | ~9 | Чипсы |
| 4 | `object_tag` | Сцена | С машиной, На море, В лесу, С цветами | ~100+ | Чипсы + поиск |

## Фильтрация через query-параметры

Фильтры передаются через URL query params. Канонический URL не меняется (SEO safe).

```
/stil/portret/?audience=devushka&occasion=den_rozhdeniya
/search?q=девушка&style=cherno_beloe&audience=para
```

| Параметр | Тип | Пример |
|----------|-----|--------|
| `audience` | string (slug) | `?audience=devushka` |
| `style` | string (slug) | `?style=portret` |
| `occasion` | string (slug) | `?occasion=svadba` |
| `object` | string (slug) | `?object=na_more` |

- Одиночный выбор — один slug на измерение
- Пустой параметр = фильтр не применён (показать все)
- На tag-странице: измерение, по которому уже отфильтровано через URL, **скрывается** из фильтров (например, на `/promty-dlya-foto-devushki/` не показывать `audience`)

## UI: плавающая кнопка фильтров

### Кнопка (свёрнутое состояние)

- **Позиция**: `fixed`, правый нижний угол, над sticky CTA-баром
- **Вид**: круглая кнопка с иконкой фильтра
- **Бейдж**: если есть активные фильтры — показать кол-во (число на кнопке)
- **Z-index**: выше грида карточек, ниже sticky CTA

```
                                    ┌─────┐
                                    │ ⚙ 2 │  ← badge с кол-вом фильтров
                                    └─────┘
```

### Панель (развёрнутое состояние)

При нажатии кнопка «разворачивается» в панель фильтров. Анимация: scale + fade из точки кнопки.

```
┌─────────────────────────────────────────────┐
│                                         ✕   │
│  Кто на фото                                │
│  [Все] [Девушка] [Мужчина] [Пара] [Семья] → │  ← горизонтальный скролл
│                                             │
│  Стиль                                      │
│  [Все] [Портрет] [Ч/Б] [3D] [Аниме] →      │  ← горизонтальный скролл
│                                             │
│  Событие                                    │
│  [Все] [ДР] [Свадьба] [НГ] [8 марта]       │
│                                             │
│  Сцена                                      │
│  🔍 [поиск...]                              │  ← поле поиска (100+ значений)
│  [На море] [В лесу] [С цветами] →           │
│                                             │
│         [ Сбросить ]  [ Применить ]          │
└─────────────────────────────────────────────┘
```

### Элементы панели

#### Заголовок измерения
- Текст: `text-sm font-medium text-zinc-500`
- Отступ снизу до чипсов: 8px

#### Чипсы фильтров
- Горизонтальный скролл (`overflow-x-auto scrollbar-none`)
- «Все» — первый чип, сбрасывает фильтр по измерению
- **Активный чип**: `bg-zinc-900 text-white`
- **Неактивный чип**: `bg-zinc-100 text-zinc-600 hover:bg-zinc-200`
- Одиночный выбор (radio-поведение)
- Размер: `px-3 py-1.5 text-sm rounded-full`

#### Сцена (object_tag) — особый случай
- Поле поиска сверху: фильтрует чипсы по введённому тексту
- Показывать max 20 чипсов, остальные — по поиску
- Placeholder: «Найти сцену...»

#### Кнопки действий
- **Сбросить** — убрать все фильтры, `text-zinc-500 hover:text-zinc-700`
- **Применить** — закрыть панель и обновить выборку, `bg-zinc-900 text-white rounded-xl`

### Анимация

- **Открытие**: панель появляется из позиции кнопки — `transform-origin: bottom right`, `scale(0.3) → scale(1)`, `opacity 0 → 1`, `200ms ease-out`
- **Закрытие**: обратная анимация, `150ms ease-in`
- **Backdrop**: затемнение фона `bg-black/20`, клик закрывает панель

### Адаптивность

| Экран | Панель фильтров |
|-------|----------------|
| Мобайл (<640px) | Bottom sheet, от нижнего края, `max-h-[70vh]`, скролл внутри |
| Десктоп (≥640px) | Popup, прижат к правому нижнему углу, `max-w-md`, `max-h-[60vh]` |

## Поведение при фильтрации

1. Пользователь нажимает кнопку фильтров → панель открывается
2. Выбирает чипсы в нужных измерениях
3. Нажимает «Применить»
4. Панель закрывается
5. Query params обновляются (`router.push` с shallow)
6. Грид сбрасывается: `offset=0`, загрузка новых карточек через API
7. На кнопке фильтров появляется бейдж с кол-вом активных фильтров

### Сброс фильтров
- Кнопка «Сбросить» в панели — убирает все query params фильтров
- Долгое нажатие / двойной клик на кнопку фильтров — быстрый сброс (P2)

## Бэкенд

### `/api/listing` — добавить query params

```typescript
// Существующие параметры
limit, offset, audience_tag, style_tag, occasion_tag, object_tag

// Новые: дублирование из query params клиента
// audience → audience_tag (merge с URL-тегом страницы)
// style → style_tag
// occasion → occasion_tag  
// object → object_tag
```

Логика: query-фильтры **добавляются** к URL-тегам страницы. Если страница `/stil/portret/` (style_tag=portret) и пользователь выбрал `audience=devushka`, то запрос: `style_tag=portret AND audience_tag=devushka`.
