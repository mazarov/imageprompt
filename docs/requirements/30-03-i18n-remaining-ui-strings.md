# Требования: вывод оставшегося UI в словари i18n (EN / RU)

**Связанный контекст:** после внедрения `next-intl` с маршрутами `/` (en, по умолчанию) и `/ru/*` часть интерфейса уже берёт строки из `landing/src/messages/{en,ru}.json`. Ниже — **договорённость по доработке** всего, что по-прежнему захардкожено на русском (или смешано) в коде.

---

## 1. Цель

- **Единый источник текстов** для пользовательского интерфейса лендинга: все видимые строки, `aria-label`, `title`, подсказки форм, сообщения об ошибках/состояниях, заголовки страниц и ключевые SEO-описания должны зависеть от **текущей локали** (`en` | `ru`).
- **Поведение:** при переключении EN/RU пользователь видит согласованный язык во **всех** экранах каталога, карточки, поиска, избранного, генераций, модалок, фильтров и юридической странице.

## 2. Область работ (in scope)

### 2.1 Страницы App Router (`landing/src/app/[locale]/…`)

| Область | Файлы (ориентир) | Что вынести |
|--------|-------------------|-------------|
| Поиск | `search/SearchResults.tsx` | Заголовки, placeholder, кнопки, пустые состояния, подсказки, sr-only |
| Каталог (теговые маршруты) | `[...slug]/page.tsx` | Хлебные крошки, подзаголовки секций, тексты блоков «как использовать» / FAQ / «ещё разделы», подписи загрузки, JSON-LD `name` для навигации — с учётом локали (`labelEn` / `labelRu` где уместно) |
| Карточка промпта | `p/[slug]/page.tsx` | `aria-label` загрузки, **generateMetadata** (title, description, OpenGraph, schema.org — не только RU), fallback-заголовки |
| Избранное | `favorites/FavoritesContent.tsx`, `favorites/page.tsx` | Все состояния: гость, пусто, CTA, метаданные страницы |
| Генерации | `generations/GenerationsContent.tsx`, `generations/page.tsx` | Аналогично + метаданные |
| Конфиденциальность | `privacy/page.tsx` | Полный текст страницы, заголовки разделов, **metadata** — для EN нужна **отдельная англоязычная редакция** (не машинный «костыль» без юридической проверки — см. п. 4) |
| Extension STV (метаданные) | `extension-stv/page.tsx`, `extension-stv/pricing/page.tsx` | `title` / `description` / canonical с учётом локали и `alternates.languages` |

### 2.2 Компоненты (`landing/src/components/`)

| Компонент | Что вынести |
|-----------|-------------|
| `CardPageClient.tsx` | Все подписи, кнопки, статусы, confirm-тексты, aria-labels, тексты про черновик/публикацию/просмотры/копирование |
| `PromptCard.tsx`, `GroupedCard.tsx`, `CategoryCard.tsx` | Fallback «Без названия», бейджи («было», черновик), «Нет фото», копирование, aria, **плюрализация** счётчиков (`N промт/промта/промтов`) — для EN использовать `next-intl` plural rules |
| `FilterPanel.tsx`, `FilterChips.tsx`, `CardFilters.tsx` | Названия измерений фильтров, заголовки панели, кнопки «Сбросить / Применить», placeholder’ы поиска, счётчики и пустые состояния |
| `InfiniteGrid.tsx`, `CategorySection.tsx` | Тексты счётчиков и «Показать все» |
| `ListingGridLoadingSkeleton.tsx` | sr-only для загрузки |
| `GenerationModal.tsx` | Заголовок, aria-labels, кнопка закрытия |
| Прочие карточные/листинговые фрагменты | Любая оставшаяся кириллица в UI (проверка grep по `[а-яА-ЯёЁ]` в `*.tsx`) |

### 2.3 Исключения и осознанные исключения

- **Поисковые подсказки / чипы** (`HomeSearch`, `SearchBar`): допускается оставить **отдельные списки** для RU и EN как данные (не обязательно одна строка в JSON), но **подписи** («Популярное», placeholder и т.д.) — только из словарей.
- **Контент из БД** (`title_ru`, `title_en`, тексты промптов): не переводить в рамках задачи; в UI — корректные **fallback**-лейблы из словарей.
- **Комментарии в коде** и внутренние **логи**: не входят в scope, если не показываются пользователю.

## 3. Технические требования

1. Использовать существующий стек: **`next-intl`**, сообщения в **`landing/src/messages/en.json`** и **`landing/src/messages/ru.json`**.
2. **Клиентские** компоненты: `useTranslations` / `useLocale` где нужно различать данные (чипы).
3. **Серверные** страницы и `generateMetadata`: `getTranslations({ locale, namespace: "…" })`; canonical и `alternates.languages` согласовать с **`src/lib/locale-path.ts`** (или общим хелпером).
4. **Плюрализация** (русские «промт/промта/промтов», английские «prompt / prompts»): через API `next-intl`, без ручных `if (n === 1)` в компонентах, где это пользовательский текст.
5. **Длинный контент** (privacy): либо вынести в markdown/MDX с локалью, либо в JSON с неймспейсом `Privacy.*` — решение за исполнителем, но **две полноценные версии** (en + ru).

## 4. Юридический и продуктовый блок

- Английская **Privacy Policy** должна быть **согласована по смыслу** с русской (не обязательно дословный перевод, но без противоречий). До юридической выверки допускается пометка «draft» в комментарии к PR или отдельный флаг в контенте — **уточнить у владельца продукта**.

## 5. Критерии приёмки

1. Поиск по репозиторию `landing/src` по шаблону **кириллицы в пользовательских строках** (JSX-текст, шаблонные строки для UI, не комментарии): **ноль** вне словарей и вне осознанно исключённых списков подсказок.
2. Ручной проход: `/`, `/ru`, каталог, карточка, поиск, избранное, генерации, фильтры, модалка генерации, privacy — язык **единообразен** с переключателем локали.
3. **`npm run build`** в `landing/` проходит без ошибок.
4. **SEO:** для ключевых страниц метаданные на EN не дублируют бездумно RU-тексты; заданы **canonical** и **hreflang**/`alternates.languages` там, где уже принято в проекте.

## 6. Приоритизация (рекомендация)

1. **P0:** Поиск, карточка (`CardPageClient` + `p/[slug]` metadata), каталог (`[...slug]` видимые строки + breadcrumbs).
2. **P1:** Карточки сетки (`PromptCard`, `GroupedCard`, `CategoryCard`), фильтры, infinite grid.
3. **P2:** Favorites, Generations, GenerationModal.
4. **P3:** Privacy (полный билингвальный контент + metadata).

## 7. Артефакты по завершении

- Обновлённые `en.json` / `ru.json` (возможно разбиение на неймспейсы по файлам — только если команда договорится).
- Краткий чеклист в PR: «пройден grep / пройден ручной smoke / build».
