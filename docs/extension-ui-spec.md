# Спецификация UI расширения Steal This Vibe (для дизайна / LLM)

> Назначение: описать **визуал**, **логику экранов** и **ограничения**, чтобы улучшать UI без разбора всего кода.  
> Для разработчиков: **`extension/DEVELOPER.md`**.

## Контекст продукта

- **Chrome Extension** (Manifest V3), **Side Panel** — узкая колонка справа (типичная ширина ~320–400 CSS px, зависит от Chrome).
- **Два визуальных слоя:**
  1. **Страница сайта** — плавающая кнопка поверх картинок (content script + Shadow DOM).
  2. **Side panel** — основной UI: авторизация, настройки генерации, превью, результаты, история.

Технически side panel: **vanilla JS**, весь UI собирается через **`innerHTML` + перерисовка `render()`** при изменении состояния (нет React/Vue в панели).

**Визуальная связка с лендингом:** тёмная панель + те же акценты, что у сайта **imageprompt.tools** — **zinc-подобные нейтрали**, градиент **indigo (`#6366f1`) → violet (`#8b5cf6`)**, шрифт **Inter**. Токены заданы в `extension/sidepanel/styles.css` как **`--stv-*`**.

## Поверхность A: кнопка на странице (content script)

| Аспект | Описание |
|--------|----------|
| **Где** | Fixed overlay, позиция у **правого верхнего** края видимой области картинки (`position: fixed` внутри shadow host). |
| **Изоляция** | **Shadow DOM** (`#stv-shadow-host`) — стили не конфликтуют с сайтом; нужно для Pinterest/React и т.п. |
| **Внешний вид** | **Мини-карточка** как в side panel: фон **zinc** `rgba(24,24,27,0.94)`, радиус **12px**, индиго-обводка/свечение; градиент **indigo → violet** только на маркере **P** (как `.stv-brand-mark`); текст **#fafafa**, бренд **#a1a1aa**; `min-height` ~46px, крупнее типографика. |
| **Текст** | Локаль по **`navigator.language`**: EN «Steal this vibe», DE «Stil übernehmen», RU «Снять стиль с фото». Если ширина картинки **&lt; ~280px** — короткий вариант (`short`), строка бренда скрыта (`.compact`). |
| **a11y** | `type="button"`, `tabindex="0"`, **`aria-label`** на языке пользователя; **`focus-visible`**: обводка `#a5b4fc`. |
| **Поведение** | Появляется при **наведении** на подходящий `<img>`. Скрытие: **нет** глобального `mouseout` (Pinterest/листинги дают ложные `mouseout` между вложенными узлами). Удержание: **`mousemove`** (throttle ~20ms) + зона **padding вокруг `getBoundingClientRect()` картинки** (`HOVER_PAD_*`, доля от размера) и запас вокруг кнопки; пока курсор в зоне — `cancelHide`. Вне зоны — `scheduleHide` (**~900ms**). `mouseleave` на `<img>` по-прежнему с проверкой перехода на Shadow host / кнопку. |
| **Клик** | Открывается side panel, в session storage кладётся `imageUrl` + контекст страницы. |

**Ограничение:** язык оверлея берётся из браузера, не из переключателя DE/RU в side panel (у панели другой origin / `localStorage`). При желании синхронизировать — писать выбор языка в `chrome.storage` из панели и читать в content script.

## Поверхность B: Side Panel — визуальный язык

Файл стилей: **`extension/sidepanel/styles.css`** (токены **`--stv-*`** в `:root`).

### Палитра (токены)

| Токен / роль | Значение (ориентир) |
|--------------|---------------------|
| Фон страницы | `--stv-bg` (`#09090b`) |
| Поверхность карточки | `--stv-surface`, бордер `--stv-border` |
| Акцент | `--stv-primary` indigo, `--stv-accent` violet |
| Текст | `--stv-text`, приглушённый `--stv-text-muted` / `--stv-text-dim` |
| Семантика | `--stv-success`, `--stv-danger`, `--stv-warning` |
| Радиусы / отступы | `--stv-radius`, `--stv-space` |

Primary-кнопки: **градиент** indigo → violet (как CTA на лендинге).

### Типографика

- **Шрифт:** Inter + системный стек
- **Заголовки карточек (`.title`):** `font-weight: 600`
- **Секции:** `.stv-section-title` — uppercase, letter-spacing (иерархия шагов)
- **Моноширинные блоки** (промпты, JSON): `.prompt-box`, `code` с фиолетовым оттенком

### Компоненты (основные классы)

- **`.stv-shell`** — вертикальный контейнер панели, max-width ~528px
- **`.stv-topbar.stv-topbar--compact`** — бренд **Prompt To Image** + **`.stv-credits-pill`** (две колонки: подпись «баланс» + число, подпись «за запуск» + стоимость в кредитах) + **`.stv-topbar-account`** (`<details>`: статус сессии, email, предупреждение о кредитах) + `.stv-tool-btn` (селект языка **ru / en / de**, выход)
- **`.card` / `.stv-card-main`** — основной поток; **`.stv-card-side`** — карточка пайплайна (левый акцент); **`.stv-card-result`**, **`.stv-card-history`**
- **`.stv-meta-strip`** — устарело (скрыто в CSS); кредиты перенесены в шапку
- **`.stv-settings-disclosure`** — свёрнутый блок параметров генерации на вкладке «Генерация»
- **`.stv-history-filter`** — фильтр записей истории (все / с изображением / только промпт)
- Баннер устаревшего промпта при смене референса до повторного extract; **indeterminate** прогресс уважает **`prefers-reduced-motion`**
- **`<section class="stv-section">`** — шаги 1–4 с **`.stv-section-head`**, **`.stv-step`** (нумерация), **`.stv-section-title`**
- **`.stv-fields` / `.stv-field` / `.stv-field-label`** — полноширинные селекты с подписями
- **`.stv-actions-primary`** — колонка основных кнопок (Сгенерировать, Купить кредиты)
- **`<details class="stv-disclosure">`** — вторичные действия (повтор / очистить / сброс); **`.stv-disclosure--dev`** — API origin + ссылка на эту спеку в репо
- **`.row`**, **`.grid`**, **`.toast`**, **`.progress-wrap`** — как раньше, стилизованы под токены

## Side Panel — логика экранов и состояний

Состояние живёт в объекте `state` (см. `sidepanel/stv-core.js`). Перерисовка: **`render()`** → загрузка / не залогинен / главный экран.

### 1. Загрузка (`state.loading`)

- **`.stv-shell`** + **`.stv-loading-card`**: марка **P**, название приложения, текст загрузки (i18n).

### 2. Не авторизован (`!state.user`)

- Тот же **`.stv-shell`** + **`.stv-topbar`** (бренд + переключатель языка).
- **`.card.stv-card-main`**: статус сессии, подсказка Google, primary **Войти**, **Проверить снова**, ошибка при необходимости.

### 3. Главный экран (`renderMain`)

Корень: **`.stv-shell`**.

1. **Topbar:** бренд **Prompt To Image** + подзаголовок (`brand_sub` i18n), селект **ru / en / de**, **Выйти**.
2. **Карточка main:**
   - Toast (если есть)
   - **`.stv-tabbar`** — три вкладки: **Промпт** (`panelTab: prompt`) | **Генерация** (`generate`) | **Мои генерации** (`history`). Активная вкладка: **`.stv-tab--active`**. Неактивные панели: **`.stv-tab-panel--hidden`**.
   - **Вкладка «Промпт»:** вертикальный **`.stv-prompt-tab-stack`**: сверху **референс** (`stv-ref-file-prompt`, **`.stv-reference-file-input`**), кнопка **«Извлечь / обновить промпт»** — полный pipeline extract → expand (и assemble при необходимости) без генерации картинки. Ниже — **промпт для генерации**: краткий текст / **«Копировать»** / по кнопке **«Редактировать блоки»** — девять полей, **«Сохранить»** — клиентская сборка через **`stv-prompt-assembly.js`**. Отдельного режима «свой текст» и override инструкции для extract в UI нет — используется серверный промпт извлечения.
   - **Вкладка «Генерация»:** сверху **`<details class="stv-generate-current-prompt">`**: подпись **«Текущий промпт…»** + в summary строка-превью; при раскрытии — полный текст в **`<pre>`** и кнопка **«Изменить»** (`#btn-edit-prompt-goto-tab`) → переключение на вкладку **«Промпт»**. Далее сетка **`.stv-compare-grid`**; grooming; прогресс; **Сгенерировать** / **Купить кредиты**; **настройки** только: модель, ratio, size, extract temperature (отдельного блока промпта в настройках нет). Сводка готово/ошибки. Блоки «Дополнительные действия» и «Для разработчиков» убраны (флаг 3× — только через `localStorage` `stv_triple_variant_flow`).
   - **Вкладка «История»:** карточка **`.stv-card-history`**; у записи превью + метаданные в строке **`.stv-history-card-row`**, раскрываемый текст промпта **`.stv-history-prompt-details--full`** на **полную ширину** карточки при открытии.

Персист: **`state.panelTab`**, **`state.promptBlocksExpanded`** (только сессия редактора — expanded не сохраняется в `stv_state_v2`, сохраняется `panelTab`).

### Условная логика (важно для UX)

- **Сгенерировать** активна только если: есть референс (`sourceImageUrl` или загруженный референс), есть хотя бы одно пользовательское фото, не идёт генерация / пауза grooming, хватает кредитов, cooldown = 0.
- **Купить кредиты** — когда не хватает кредитов и не идёт генерация; при ожидании оплаты — другой текст кнопки.
- **Очистить результат** — сбрасывает vibe/prompts/results, **не** сбрасывает загруженное фото с ПК.
- **Сбросить сессию** — полный сброс включая storage path и локальный state key.

### Фоновые обновления

- Каждые **30 s** тихая проверка auth/credits
- При **visibilitychange** → снова refresh
- После оплаты в Telegram — polling кредитов

## Локализация

- Файл `sidepanel/i18n.js`: ключи **RU** и **DE** (в т.ч. `section_photos_compare`, `compare_col_your_photo`, `compare_col_reference`, `compare_photo_empty`, `field_*`, …).
- Выбор языка: `navigator.language` начинается с `de` → DE, иначе RU; ручной переключатель → `localStorage.stv_ui_lang`.
- **Кнопка на странице** и часть toast/сообщений сохранения могут быть **не полностью** переведены — см. код.

## Ограничения для дизайна

1. **Узкая ширина** — основной поток колонкой; вторичные действия вынесены в `<details>`.
2. **Полный ре-рендер DOM** при каждом `render()` — сложные анимации/фокус либо не полагаться на сохранение, либо выносить в неперерисовываемые узлы (сейчас не сделано).
3. **CSP расширения** — inline scripts в HTML ограничены; стили панели в `styles.css`.
4. **Нет светлой темы** в панели — только тёмная (согласовано с привычным Chrome side panel).
5. **Без UI-kit** — новый визуал: токены в CSS + разметка в `app.js`.
